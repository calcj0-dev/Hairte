# Hairte 仕様書

## 1. アプリ概要

- **アプリ名**: Hairte
- **名前の由来**: Hair（ヘアー）+ カルテ
- **目的**: 美容室でのオーダーに困るユーザーのために、髪のカルテ（カット履歴・髪の特徴）を記録・管理し、次回のオーダーをスムーズにする
- **対象ユーザー**: 美容室に通う一般ユーザー（将来的には美容師側にも展開予定）
- **プラットフォーム**: スマートフォンアプリ（iOS / Android）

---

## 2. 機能要件

### 2.1 主要機能

| 機能名 | 概要 | 優先度 |
|--------|------|--------|
| 髪カルテ | 自分の髪の特徴を記録・管理する | 高 |
| カット履歴 | 過去の美容室訪問履歴を記録する | 高 |
| オーダー支援 | 伝え方テンプレート・参考画像保存（カット履歴に統合） | 低 |
| 通知機能 | 定期的なリマインダーを送る | 高 |
| 検索・フィルター | 履歴を条件で絞り込む | 高 |
| アカウント管理 | ログイン・ログアウト・退会（データ即時削除）を行う | 高 |
| 写真管理 | before / after の写真を保存・閲覧する | 中 |
| 美容室管理 | よく行く美容室・担当美容師を登録する | 中 |
| SNSシェア | ヘアスタイルをSNSにシェアする | 中 |

### 2.2 機能詳細

#### 機能1: 髪カルテ
- 説明: 自分の髪の特徴を項目ごとに記録しておく機能
- 記録項目:
  - 髪質（硬い / 普通 / 柔らかい）
  - 髪の太さ（太い / 普通 / 細い）
  - 髪の量（多い / 普通 / 少ない）
  - くせの強さ（直毛 / 少しくせ / くせ強め）
  - ダメージ具合（健康 / 少しダメージ / ダメージあり）
  - 頭の形・生え癖（自由記述）
  - その他メモ

#### 機能2: カット履歴
- 説明: 美容室に行くたびに記録を残す機能
- 記録項目:
  - 日付
  - 美容室名
  - 担当美容師名
  - オーダー内容（メニュー・スタイル）
  - 仕上がりの満足度（5段階）
  - before / after 写真
  - メモ（気になったこと・次回への引き継ぎ）

#### 機能3: オーダー支援（サブ機能・カット履歴に統合）
- 説明: 美容師への伝え方をサポートするサブ機能。独立した画面は持たず、カット履歴に統合する
- 統合先:
  - **カット履歴登録画面**: オーダー内容入力欄の近くに「テンプレートを使う」ボタンを配置。選択するとオーダー内容欄に反映
  - **カット履歴詳細画面**: 参考スタイル画像の保存・閲覧セクションを追加
- **プロトタイプ方針**: テンプレートは固定文言で実装する。動的生成・AI活用は本リリース以降に検討

#### 機能4: 写真管理
- 説明: ヘアスタイルの写真を一覧で確認できる機能
- 内容:
  - カット履歴に紐づいた before / after 写真（各1枚）
  - 参考にしたいスタイル画像の保存
- **プロトタイプ方針**: 写真の枚数・容量制限なし。本リリース時にFirebase Storageのコストを踏まえて上限を設定する

#### 機能5: 通知機能
- 説明: 美容室に行くタイミングをリマインドする機能
- 内容:
  - 前回のカットから指定した期間が経過したら通知（例: 1ヶ月・2ヶ月・3ヶ月）
  - 通知のON / OFF・間隔はユーザーが設定可能

#### 機能6: 検索・フィルター
- 説明: カット履歴を条件で絞り込む機能
- 検索条件:
  - 美容室名
  - 日付（期間指定）
  - 満足度
  - 担当美容師名

#### 機能7: SNSシェア
- 説明: ヘアスタイルの写真やオーダー内容をSNSにシェアする機能
- 対応SNS: Instagram、X（Twitter）、LINE など
- シェア内容: after写真 ＋ オーダー内容のテキスト

#### 機能8: 美容室管理
- 説明: よく行く美容室を登録・選択する機能
- 記録項目:
  - 美容室名
  - 住所 / 電話番号
  - メモ
- **注記**: 独立した管理画面は持たない。カット履歴登録時の「美容室を選択」からのみアクセス。編集・削除はプロトタイプ対象外

#### 機能9: アカウント管理
- 説明: ログイン・ログアウト・退会を行う機能
- 内容:
  - Googleログイン / Appleログイン（Firebase Auth）
  - ログアウト
  - 退会（アカウント削除）
    - Firestore上のユーザーデータ（カルテ・履歴・美容室）を即時削除
    - Firebase Storage上の写真を即時削除
    - Firebase Authのアカウントを即時削除
    - 退会前に確認ダイアログを表示する

## 3. 非機能要件

- **パフォーマンス**: 画面遷移は1秒以内
- **セキュリティ**: ユーザーデータはアカウントに紐づけて管理、他ユーザーからアクセス不可
- **可用性**: 99%以上の稼働率
- **スケーラビリティ**: 将来的な美容師側機能の追加を考慮した設計
- **プライバシー**: プライバシーポリシー・利用規約の整備（優先度低・リリース前に対応）

## 4. 画面設計

### 4.1 画面一覧

| 画面名 | 概要 | 遷移元 |
|--------|------|--------|
| ホーム | 直近の履歴を表示 | 起動時 |
| 髪カルテ | 自分の髪の特徴を記録・編集 | BottomNav |
| カット履歴一覧 | 過去の訪問履歴を一覧表示 | BottomNav |
| カット履歴詳細 | 個別の訪問履歴を表示・参考画像管理 | カット履歴一覧 |
| カット履歴登録 | 新しい訪問履歴を登録・テンプレート参照 | カット履歴一覧 |
| 美容室一覧 | 登録済みの美容室を一覧表示 | カット履歴登録 |
| 設定 | 通知設定・ログアウト・退会 | BottomNav |

### 4.2 画面遷移

```
起動
 └── ホーム（BottomNav）
      ├── [タブ] 髪カルテ
      ├── [タブ] カット履歴一覧
      │    ├── カット履歴詳細
      │    │    └── 参考スタイル画像の管理（オーダー支援・統合）
      │    └── カット履歴登録
      │         ├── テンプレート選択シート（オーダー支援・統合）
      │         └── 美容室一覧（美容室選択）
      └── [タブ] 設定
```

## 5. データ設計

### 5.1 データモデル

#### User（ユーザー）
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | String | ユーザーID |
| name | String | 名前 |
| email | String | メールアドレス |
| createdAt | DateTime | 登録日時 |

#### HairKarte（髪カルテ）
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | String | カルテID |
| userId | String | ユーザーID |
| texture | String | 髪質 |
| thickness | String | 髪の太さ |
| volume | String | 髪の量 |
| curl | String | くせの強さ |
| damage | String | ダメージ具合 |
| headShape | String | 頭の形・生え癖（自由記述） |
| memo | String | その他メモ |
| updatedAt | DateTime | 更新日時 |

#### CutHistory（カット履歴）
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | String | 履歴ID |
| userId | String | ユーザーID |
| salonId | String | 美容室ID |
| date | Date | 訪問日 |
| stylist | String | 担当美容師名（自由記述） |
| order | String | オーダー内容 |
| satisfaction | Int | 満足度（1〜5） |
| memo | String | メモ |
| beforePhoto | String | before写真URL |
| afterPhoto | String | after写真URL |
| createdAt | DateTime | 登録日時 |

> **注記**: `stylist` は MVP では自由記述の文字列とする。将来的には `Stylist` エンティティ（`id`, `salonId`, `name`, `memo`）を追加し、`stylistId` で参照する形への移行を検討する。

#### Salon（美容室）
| フィールド | 型 | 説明 |
|------------|-----|------|
| id | String | 美容室ID |
| userId | String | ユーザーID |
| name | String | 美容室名 |
| address | String | 住所 |
| phone | String | 電話番号 |
| memo | String | メモ |

## 6. ビジネスモデル

### 6.1 プロトタイプ方針

- **プロトタイプフェーズ（現在）**: 全機能・全ユーザーに無料で提供。広告・課金機能なし
- **本リリース以降（将来）**: 広告表示（無料）＋広告非表示（サブスクリプション）の構成を検討

### 6.2 将来のプラン構成（暫定）

| プラン | 価格 | 内容 |
|--------|------|------|
| 無料プラン | 無料 | 全機能利用可・広告表示あり |
| プレミアムプラン | 未定（月額 or 年額） | 全機能利用可・広告非表示 |

### 6.3 収益化の方針（将来）
- 広告実装: Google AdMob等を検討。詳細はプロトタイプ検証後に決定
- 課金: App Store / Google Playのアプリ内課金（IAP）を使用
- 無料/プレミアムの機能境界: 現状未定義（優先度低）

## 7. 技術スタック

- **フロントエンド**: Flutter（Dart）
- **バックエンド**: Firebase
- **データベース**: Firestore
- **認証**: Firebase Auth（Googleログイン / Appleログイン）
- **ストレージ**: Firebase Storage（写真保存用）
- **通知**: Firebase Cloud Messaging（FCM）
